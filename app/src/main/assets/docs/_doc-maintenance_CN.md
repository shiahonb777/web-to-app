# 文档维护指南

> **当项目文档与实际代码出现偏差时，按本指南执行审计与纠偏。** 本文档定义了文档审计的触发条件、执行流程和纠偏方法。

---

## 一、什么时候需要审计

文档不会自己过时，但代码会持续演进。以下情况应触发文档审计：

| 触发条件 | 说明 |
|---------|------|
| 大规模重构后 | 目录结构、模块划分、核心类名发生变动 |
| 新功能上线后 | 新增了重要模块或功能，但文档未同步更新 |
| 定期巡检 | 建议每 1-2 个月执行一次轻量巡检 |
| 发现文档描述与实际行为不符 | 任何人发现即可触发 |

---

## 二、审计流程

```
1. 盘点文档 → 2. 逐文档验证 → 3. 记录偏差 → 4. 纠偏 → 5. 验证纠偏结果
```

### Step 1: 盘点文档

列出 `docs/` 目录下所有文档及其声明的主题：

```bash
find docs/ -name "*.md" ! -name "_*" | sort
```

排除以 `_` 开头的元文档（`_doc-manual.md`、`_doc-maintenance.md`），它们是规范本身，不需要对标代码验证。

### Step 2: 逐文档验证

对每个文档，执行以下检查：

| 检查项 | 验证方法 |
|--------|---------|
| **文件路径引用** | 文档中引用的每个文件路径，用 `ls` 或 `find` 确认文件是否存在 |
| **类名/函数名/属性名引用** | 用 `grep` 确认引用的符号在代码中是否存在 |
| **架构描述** | 对比实际目录结构与文档描述是否一致 |
| **流程描述** | 对照代码逻辑，确认文档描述的流程与实际代码行为一致 |
| **规则/约束声明** | 检查文档声明的规则是否仍被代码遵守（如命名规范、代码模式等） |
| **命令/脚本可用性** | 文档中给出的 grep/构建命令，实际执行确认是否仍有效 |
| **摘要行准确性** | 文档开头的 `>` 摘要是否仍准确描述文档内容 |

### Step 3: 记录偏差

对每个文档，记录发现的偏差：

```markdown
### [文档名].md 偏差记录

| 偏差位置 | 文档描述 | 实际情况 | 纠偏方式 |
|---------|---------|---------|---------|
| 第二节-核心文件表 | 引用 `core/auth/AuthRepo.kt` | 实际路径为 `core/auth/AuthRepository.kt` | 更新路径 |
| 第三节-流程描述 | "登录后自动刷新 Token" | 已改为手动刷新 | 更新描述 |
```

### Step 4: 纠偏

按 `_doc-manual.md` 的规范修改文档：

1. 更新过时的引用（路径、类名、属性名）
2. 更新过时的描述（流程、行为、规则）
3. 删除已失效的内容（已不存在的模块、已废弃的流程）
4. 补充缺失的内容（新模块、新功能未被文档覆盖）
5. 追加修订记录

### Step 5: 验证纠偏结果

纠偏完成后，对修改过的文档再次执行 Step 2 的检查，确认偏差已消除。

## 三、各文档的专项审计要点

不同文档有不同关注点，审计时应重点检查：

### `project-overview.md`

| 检查项 | 验证命令 |
|--------|---------|
| 目录结构是否与实际一致 | `find app/src/main/java/com/webtoapp/ -type d -maxdepth 3 \| sort` |
| 核心模块表中的路径是否存在 | `ls -d <文档中每个路径>` |
| 代码规模描述是否大致准确 | `find app/src/main/java -name "*.kt" \| xargs wc -l \| tail -1` |
| 文档索引表是否包含所有文档 | `find docs/ -name "*.md" ! -name "_*" \| sort` |

### `i18n-localization-guide.md`

| 检查项 | 验证命令 |
|--------|---------|
| 核心文件路径是否存在 | `ls <Strings.kt路径> <AppLanguage.kt路径>` |
| 支持语言列表是否与 `AppLanguage` 枚举一致 | `grep "enum class AppLanguage" -A 10 <AppLanguage.kt路径>` |
| grep 扫描命令是否仍能执行 | 实际执行文档中的 grep 命令 |
| 代码示例中的属性名是否在 `Strings.kt` 中存在 | `grep "val communityPostsLoadFailed" <Strings.kt路径>` |
| 判断规则表中的模式是否仍存在于代码中 | 抽样 grep 验证 |

## 四、常见偏差模式

| 偏差类型 | 典型表现 | 预防措施 |
|---------|---------|---------|
| **路径漂移** | 文件被移动/重命名，文档仍引用旧路径 | 重构时同步搜索文档中的引用 |
| **符号漂移** | 类/函数/属性被重命名，文档仍用旧名 | 重命名时同步搜索文档 |
| **流程漂移** | 代码逻辑被修改，文档描述未更新 | 修改核心流程时检查是否有对应文档 |
| **结构漂移** | 新增/删除了目录或模块，文档未同步 | 定期巡检 |
| **命令失效** | grep 模式或构建命令因代码变化而失效 | 审计时实际执行验证 |
